O Problema
Violência contra a mulher no Brasil
Milhões de mulheres vivem em situação de risco sem saber avaliar a gravidade ou quais passos tomar. A falta de acesso a ferramentas especializadas agrava a vulnerabilidade.
1,4mi
registros de violência doméstica no Brasil em 2023
66%
das vítimas não buscam ajuda especializada
⚠️ Desafio identificado
Não existe ferramenta mobile acessível que aplique a metodologia científica AR PAX/FRIDA para avaliar o risco de forma estruturada e orientar a vítima.
A Solução
App de Avaliação de Risco
Um aplicativo móvel que implementa fielmente a metodologia AR PAX/FRIDA, utilizada pelo programa Desperte Mulher, guiando a usuária por um questionário estruturado e gerando uma classificação de risco personalizada.
Flutter Dart 3.11 Android & iOS Offline Código aberto
  • Questionário de 5 etapas com perguntas validadas cientificamente
  • Algoritmo fiel à planilha oficial AR PAX/FRIDA (pesos e matriz de risco)
  • 5 níveis de risco: Muito Baixo → Extremo
  • Recomendações personalizadas por nível de risco
  • Destaque automático para emergências (180 / 190) em risco Alto e Extremo
  • Funciona 100% offline — sem envio de dados a terceiros

Decisões técnicas
  • Flutter 3.41 + Dart 3.11 — multiplataforma com uma única base de código
  • JSON em assets — questões carregadas offline, sem dependência de API
  • Cache por página — estado do quiz preservado ao navegar entre etapas
  • ThemeData centralizado — identidade visual consistente em todas as telas
  • GlobalKey + Form — validação robusta no login sem re-renders desnecessários
  • 3 testes automatizados — validam o algoritmo de risco contra resultados esperados
Qualidade do Código
Problemas identificados
e corrigidos
  • late BuildContext em LoginPage e ProfilePage — crash potencial
  • Login sem nenhuma validação de formulário
  • Tela de cadastro sem feedback ao usuário
→ APÓS REFATORAÇÃO
  • Contexto usado diretamente do build()
  • Validação de e-mail e senha mínima de 6 chars
  • SnackBar de sucesso no cadastro
Antes BUG
class LoginPage extends StatelessWidget { late BuildContext pageContext; // ← ANTI-PATTERN void _onLoginPressed() { Navigator.pushNamed(pageContext, ...); // pode crashar } }
Depois CORRIGIDO
class LoginPage extends StatefulWidget { void _onLoginPressed() { if (_formKey.currentState!.validate()) { Navigator.pushNamed(context, ...); // ← seguro } } }
Nossa Proposta de Layout
Por que escolher
nosso layout?
O layout original usa Material Design padrão sem identidade visual. Nosso layout foi projetado com foco no contexto de uso: uma mulher em situação de vulnerabilidade merece uma interface acolhedora e clara.
Layout original
  • Sem identidade visual
  • AppBar genérica "Login"
  • Sem validação visível
  • Cores padrão do Material
  • Emergência sem destaque
Nosso layout
  • Logo + nome do app
  • Paleta roxa coerente com o tema
  • Validação com mensagens claras
  • Tema global centralizado
  • Banner vermelho de emergência
🎯 Argumento central
Design é parte da solução. Para uma mulher em risco, uma interface clara e empática pode ser a diferença entre pedir ajuda ou não.
Testes Automatizados
3 testes · 100% passando
Os testes validam o algoritmo de risco contra resultados esperados, garantindo que nenhuma mudança de layout quebre a lógica de negócio.
$ flutter test +1: quiz_low_risk_test — Muito Baixo (20%) +2: quiz_high_risk_test — Extremo (100%) +3: quiz_back_button — estado preservado All tests passed!
O que cada teste valida
🧪 quiz_low_risk_test
Responder "Não" a tudo → risco Muito Baixo (20%). Valida o limite inferior da matriz.
🧪 quiz_high_risk_test
Resposta mais grave em tudo → risco Extremo (100%). Validado por simulação independente em Python.
🧪 quiz_back_button_test
Navegar para etapa anterior e avançar preserva as respostas já selecionadas.
